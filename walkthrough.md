# Phase 3: Inventory & Warehouse Management — Walkthrough

## ما تم بناؤه

### Backend

#### `inventoryController.ts` (جديد)
كنترولر متكامل يدعم كل عمليات المخزون:

| الوظيفة | Endpoint | الوصف |
|---------|---------|-------|
| `getUOMs` / `createUOM` / `updateUOM` | `GET/POST/PUT /api/inventory/uoms` | إدارة وحدات القياس مع دعم تحويل الوحدات |
| `getCategories` / `createCategory` / `updateCategory` | `GET/POST/PUT /api/inventory/categories` | تصنيفات هرمية للأصناف |
| `getWarehouses` / `createWarehouse` / `updateWarehouse` | `GET/POST/PUT /api/inventory/warehouses` | مستودعات متعددة مرتبطة بالفروع |
| `getItems` / `createItem` / `updateItem` | `GET/POST/PUT /api/inventory/items` | دليل الأصناف مع ربط الحسابات المحاسبية |
| `getInventoryBalances` | `GET /api/inventory/balances` | أرصدة المخزون بالكمية والتكلفة والقيمة |
| `getTransactions` | `GET /api/inventory/transactions` | قائمة الحركات المخزنية |
| `createInventoryTransaction` | `POST /api/inventory/transactions` | ترحيل حركة مخزنية جديدة |

#### منطق المتوسط المرجح المتحرك (WAC)
عند كل استلام، يُعاد حساب متوسط التكلفة:
```
newAvgCost = (currentQty × currentAvgCost + receivedQty × unitCost) / newTotalQty
```
عند كل صرف أو بيع، تُستخدم التكلفة المتوسطة الحالية تلقائياً.

#### القيود المحاسبية التلقائية
تُنشأ قيود محاسبية **فقط** للعمليات ذات الأثر المالي:

| نوع الحركة | الأثر المحاسبي |
|-----------|--------------|
| ✅ استلام مشتريات (Receipt) | مدين: حساب المخزون / دائن: حساب إزاء |
| ✅ صرف مخزون (Issue) | مدين: تكلفة البضاعة المباعة / دائن: حساب المخزون |
| ✅ رصيد افتتاحي (Opening) | مدين: حساب المخزون / دائن: حساب إزاء |
| ✅ تسوية مخزون (Adjustment) | يُحدد حسب طبيعة التسوية |
| 🚫 تحويل بين مستودعات (Transfer) | لا قيد محاسبي — تحويل داخلي فقط |

#### ميزات `InventoryTransactions.tsx`
- إحصائيات سريعة لكل نوع حركة
- تنبيه واضح عند الحركات ذات الأثر المالي (سيتم إنشاء قيد محاسبي)
- تنبيه آخر للتحويلات الداخلية (بدون قيد)
- إدخال متعدد الأصناف في حركة واحدة
- دعم رقم الدفعة (Batch) وتاريخ الانتهاء
- تكلفة الوحدة تُملأ تلقائياً من بيانات الصنف
- التحقق من توفر الكمية قبل الصرف

---

## ما يجب التحقق منه

> [!IMPORTANT]
> **خطوات التحقق اليدوي:**
> 1. إنشاء وحدة قياس → تصنيف → مستودع → صنف (مع ربط حساب المخزون)
> 2. إنشاء حركة **رصيد افتتاحي** للصنف
> 3. التحقق من `أرصدة المخزون` — يجب أن تظهر الكمية والتكلفة
> 4. التحقق من `تحليل المعاملات المالية` — يجب أن يظهر قيد محاسبي تلقائي
> 5. إنشاء **تحويل بين مستودعات** والتحقق من عدم إنشاء قيد محاسبي
